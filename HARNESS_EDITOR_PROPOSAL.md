# Harness Editor

**AI 코딩 에이전트를 산업 현장에 맞게 깎는 노코드 에디터**

---

## 1. 문제 인식

### AI 코딩 에이전트의 시대가 열렸다

2025년, Anthropic의 Claude Code, OpenAI의 Codex CLI, OpenCode 등 AI 코딩 에이전트가 폭발적으로 성장하고 있다. 이들은 코드를 읽고, 쓰고, 테스트하고, 배포까지 자동화하는 범용 에이전트다.

그러나 **범용 에이전트를 산업 현장에 바로 투입하면 실패한다.**

금융권에서 AI 에이전트가 고객 데이터가 담긴 파일을 무심코 수정하면? 의료 소프트웨어에서 검증 없이 코드를 배포하면? 방위산업에서 보안 등급을 무시하고 외부 API를 호출하면?

### "하네스(Harness)"라는 해답

업계에서는 이 문제를 **하네스**로 풀고 있다. 하네스는 AI 에이전트 위에 올려놓는 제어 장치다 — 어떤 도구를 쓸 수 있는지(권한), 어떤 순서로 일하는지(워크플로우), 실수했을 때 어떻게 복구하는지(안전장치)를 정의한다.

현재 GitHub에서 인기를 끌고 있는 오픈소스 하네스들:

| 하네스 | 대상 런타임 | GitHub Stars | 특징 |
|---|---|---|---|
| **oh-my-claudecode** | Claude Code | 5,800+ | 39개 스킬, 21개 에이전트, 10개 훅 |
| **oh-my-openagent** | OpenCode | 3,200+ | 52개 훅, 3-tier MCP, Boulder 지속성 |
| **ouroboros** | Claude Code | 2,400+ | Seed 기반 워크플로우 엔진, 진화 루프 |
| **superpowers** | 6개 플랫폼 | 4,100+ | 행동 지침 스킬팩, 6개 런타임 지원 |
| **gstack** | 8개 플랫폼 | 3,000+ | 역할 기반 스프린트 방법론 |
| **oh-my-codex** | Codex CLI | 1,500+ | 40개 스킬, Codex 네이티브 훅 통합 |

### 진짜 문제: 범용 하네스는 범용이다

이 하네스들은 **개발자가 개발자를 위해** 만든 범용 도구다. 마크다운 파일을 직접 작성하고, 훅 스크립트를 Node.js로 코딩하고, YAML 설정을 손으로 편집해야 한다.

그러나 AI를 도입하려는 실제 산업 현장의 상황은 다르다:

- **금융**: 특정 파일 수정 시 반드시 시니어 리뷰어 승인 필요. 외부 API 호출 차단. 감사 로그 자동 기록.
- **의료**: HIPAA 규정 준수 체크가 모든 코드 변경에 선행. 환자 데이터 접근 시 2단계 승인.
- **제조**: PLC 제어 코드 변경 시 시뮬레이션 통과 필수. 안전 등급별 차등 권한.
- **법률**: 계약서 템플릿 수정 시 법무팀 승인 워크플로우. 기밀 문서 접근 제어.
- **교육**: 학생 제출 코드에 대한 자동 피드백 파이프라인. 채점 기준별 평가 훅.

이들은 **자기 산업에 맞는 커스텀 하네스**가 필요하지만, `.mjs` 훅 스크립트를 작성할 역량도, 시간도 없다.

## 2. 솔루션: Harness Editor

**Harness Editor는 AI 코딩 에이전트의 행동을 시각적으로 설계하는 노코드 에디터다.**

드래그앤드롭으로 블록을 조합하거나, AI 채팅으로 "이런 규칙 추가해줘"라고 말하면, 에디터가 실행 가능한 하네스 패키지를 자동으로 생성한다.

### 핵심 메타포

```
Figma가 디자이너에게 해준 것을,
Harness Editor가 AI 에이전트 관리자에게 해준다.

"코드를 몰라도 AI 에이전트의 행동을 설계할 수 있다."
```

### 작동 방식

```
┌─────────────────────────────────────────────────────────┐
│                    Harness Editor                        │
│                                                         │
│  ┌──────────┐  ┌──────────────────────┐  ┌───────────┐ │
│  │ 블록     │  │   노드 그래프 캔버스    │  │ 인스펙터  │ │
│  │ 카탈로그  │  │                      │  │           │ │
│  │          │  │  [SessionStart]       │  │ 스킬 편집  │ │
│  │ 원자:    │  │       │              │  │ 속성 설정  │ │
│  │ - 훅     │  │       ▼              │  │ 호환성    │ │
│  │ - 조건   │  │  [PreToolUse]        │  │           │ │
│  │ - 스킬   │  │       │              │  │           │ │
│  │ - 상태   │  │   ┌───┴───┐         │  │           │ │
│  │          │  │   ▼       ▼         │  │           │ │
│  │ 패턴:    │  │ [Review] [Block]     │  │           │ │
│  │ - 권한   │  │                      │  │           │ │
│  │ - 루프   │  └──────────────────────┘  └───────────┘ │
│  │ - 지속   │                                          │
│  └──────────┘  ┌──────────────────────┐                │
│                │    AI 채팅 패널       │                │
│                │ "Write 도구 사용 시   │                │
│                │  리뷰 스킬 실행하게   │                │
│                │  해줘"               │                │
│                │  → [노드 자동 생성]   │                │
│                └──────────────────────┘                │
│                                                         │
│  ┌─────────────────────────────────────────────────────┐│
│  │ Export: [Claude Code ✓] [OpenCode ✓] [Codex ✓]     ││
│  │ Test:   [라이브 샌드박스 실행] → 노드 실시간 하이라이트  ││
│  └─────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘
```

### 두 가지 입력 모드

**1. 드래그앤드롭 (비주얼 모드)**
- 블록 카탈로그에서 원하는 훅/스킬/조건 블록을 캔버스로 드래그
- 블록 간 와이어로 연결하여 워크플로우 구성
- 복합 패턴(Permission Gate, Review Loop 등)은 원클릭 배치 후 더블클릭으로 내부 편집

**2. AI 채팅 (대화 모드)**
- "migration 파일 수정 시 시니어 리뷰 필수로 해줘"
- AI가 자연어를 해석하여 적절한 노드를 자동 생성 및 와이어링
- 캔버스에 실시간으로 노드가 나타나며 시각적 피드백 제공
- 카탈로그에 없는 커스텀 행동도 AI가 새 블록을 즉석 생성

### 컴파일 & 테스트

**Export** — 설계한 하네스를 실제 배포 가능한 패키지로 변환:
- Claude Code → `plugin.json` + `SKILL.md` + `.mjs` 훅 스크립트
- OpenCode → `opencode.jsonc` + TypeScript 플러그인 모듈
- Codex → `.codex/hooks.json` + 스킬 팩

**Test** — 설계한 하네스가 실제로 의도대로 동작하는지 검증:
- 라이브 샌드박스에서 실제 AI 에이전트 세션 실행
- 프롬프트 입력 시 노드 그래프가 실시간으로 반짝이며 훅 발동/스킬 호출/상태 변화를 시각화
- 핫리로드 지원 — 노드 수정 즉시 반영, 재시작 없이 재테스트

- 공공/국방: 보안 등급 기반 접근 제어 하네스 필요

**Secondary: AI 에이전트 커스터마이저**

- DevOps/Platform 팀: 사내 AI 에이전트 표준 하네스 관리
- 교육기관: 학습/채점 파이프라인 하네스

### 경쟁 환경

| 구분 | 기존 하네스 (omc, omo 등) | Harness Editor (우리) |
|---|---|---|
| 대상 사용자 | 개발자 | 비개발자 포함 |
| 편집 방식 | 마크다운 + 코드 직접 작성 | 노코드 비주얼 에디터 |
| 커스터마이징 | 포크 후 코드 수정 | 드래그앤드롭 + AI 채팅 |

**직접 경쟁자는 현재 존재하지 않는다.** AI 에이전트 하네스의 "비주얼 에디터"라는 카테고리 자체가 새롭다.

### 블록 카탈로그 (Day 1)

**원자 블록 (11종)**
- 훅 이벤트: SessionStart, PreToolUse, PostToolUse, UserPromptSubmit, Stop
- 플로우: Condition, Loop, Merge, Sequence
- 데이터: StateRead, StateWrite
- 통합: Skill, Agent, MCPServer, SystemPrompt

**복합 패턴 (9종)** — 유명 오픈소스 하네스에서 추출
- Permission Gate (omc 유래): 도구 사용 전 권한 체크
- Review Loop (omc/omo 유래): 리뷰 실패 시 자동 재시도
- Session Init Bundle (전체 유래): 세션 시작 시 초기화 번들
- Boulder Continuation (omo 유래): 세션 간 상태 유지
- Ralph Loop (omo/ouroboros 유래): 평가-변이-재시도 진화 루프
- Subagent Delegation (omc/omo 유래): 서브에이전트 위임
- 3-Tier MCP Registration (omo 유래): 계층적 MCP 도구 등록
- Lore/Memory Persist (omx 유래): 학습 추출 및 영속 저장
- Evolutionary Seed (ouroboros 유래): Seed 기반 진화 파이프라인

## 7. 팀 역량

### 왜 우리인가

이 제품의 핵심 장벽은 **"하네스가 실제로 어떻게 작동하는지"에 대한 깊은 이해**다.

우리는 시중의 주요 오픈소스 하네스 6개(ouroboros, oh-my-claudecode, oh-my-openagent, oh-my-codex, superpowers, gstack)를 모두 클론하여 소스코드를 심층 분석했다. 각 하네스의 훅 라이프사이클, 상태 관리 패턴, MCP 통합 방식, 스킬 구성 메커니즘, 배포 파이프라인을 코드 레벨에서 완전히 이해하고 있다.

이 분석에서 도출한 핵심 인사이트:

1. **SKILL.md + YAML frontmatter는 사실상 업계 표준** — 6개 하네스 모두 사용
2. **런타임 훅의 깊이가 가장 큰 차별화 요소** — 하네스의 복잡도를 결정
3. **3가지 훅 실행 모델**(프로세스/인프로세스/RPC)이 존재 — 추상화 레이어 필요
4. **코드 생성이 핵심 기술** — 비주얼 노드 → 실행 가능한 `.mjs`/`.ts` 코드 변환

이 지식이 Harness Editor의 IR 설계, 컴파일러 아키텍처, 블록 카탈로그 구성의 기반이 된다.

## 8. 핵심 가치 제안
